# НейроВітамінка 365

## Що це

Річна цифрова програма розвитку дитини 3–7 років у форматі щоденних 10-хвилинних «пригод». Кожен день — аудіо-казка + відео-вправа від нейропсихолога, об'єднані в одну серіальну історію з персонажем Паростком. Наукова база — нейропсихологічна система А. Лурії (розвиток через рух та емоції).

## Для кого

**Основна аудиторія:** батьки дітей 3–7 років, які хочуть системно розвивати дитину, але перевантажені хаотичним контентом і відчувають тривогу через «цифровий шум».

**Вторинна аудиторія:** самі діти — взаємодіють разом з батьками.

**Больова точка:** батьки хочуть займатися з дитиною, але не знають що, як і в якому порядку. Застосунок знімає цю тривогу: план вже є, треба лише натиснути одну кнопку.

## Структура одного дня (4 фази)

### Фаза 0 — Початок дня
- Вхід у застосунок, вітання Паростка
- Якщо день пропущено — окрема поведінка (див. нижче)

### Фаза 1 — Казка (3–5 хв)
- Текст казки + аудіо-програвач
- Аудіоплеєр як ігровий об'єкт: іконка Паростка рухається «стежкою» від початку до кінця казки
- Кнопка Play оформлена як «Чарівна зернина» що проростає
- Приховані інтерактивні зони (2–3 секретні дотики): наприклад, листочок у куті здригається і з нього падає крапля роси
- Перехід: кнопка «До вправи»

### Фаза 2 — Відео-вправа (1,5–3 хв)
- Локація: галявина, де Паросток вітається і виконує вправу
- Структура відео:
  - 0:00–0:30 — коротке пояснення вправи
  - 0:30–2:00 — виконання разом з персонажем
  - 2:00–2:30 — похвала дитини, заклик до спокою
- Відео вмикається тапом на екран

### Фаза 3 — Завершення дня
- Похвала батькам: що саме прокачали і чому це важливо для мозку
- Нагорода: +1 листочок до віртуального садку
- Анонс наступного дня

## Питання, які потрібно вирішити до розробки

### ❓ Пропущений день
Що відбувається, якщо відкрити застосунок після перерви?
- **Варіант А:** Продовження з наступного дня (день X ≠ реальний день)
- **Варіант Б:** Можливість «нагнати» пропущене (до 3 днів)
- **Варіант В:** Стрік ламається, але без штрафу — просто продовжують

> Рішення потрібне — від нього залежить архітектура.

### ❓ Карта/календар днів
- Чи бачить користувач всі 365 днів наперед?
- Чи можна повернутись до вже пройдених днів і переслухати казку?

### ❓ Вік 3–7 — чи адаптується контент?
Трирічна і семирічна дитина — принципово різний рівень.
- Якщо контент адаптується — це фактично кілька паралельних програм
- Якщо ні — навіщо питати вік при реєстрації?

### ❓ Що після 365-го дня?
- Нова програма? Повтор? Бібліотека казок?

## Екрани

### MVP (поточна версія)

| Екран | Опис |
|-------|------|
| **Онбординг** | Темний екран → дощ → Паросток проростає → привітання → ім'я та вік дитини |
| **Головний екран** | Вітання по імені, день X з 365, прогрес-бар, кнопка «Пригода дня», міні-садок |
| **Екран пригоди** | Казка → вправа → завершення з нагородою |

### Наступні версії

- Аудіоплеєр як ігровий об'єкт з Паростком
- Карта/календар 365 днів
- Бібліотека пройдених казок
- Приховані інтерактивні зони в казці
- Профіль дитини з графіком розвитку
- Аналітика та тести (координація, увага)
- База знань для батьків (статті)
- Друковані матеріали (лабіринти, розмальовки)
- Push-сповіщення

---

## Чого НЕ буде в MVP

- Реєстрація та авторизація (немає backend)
- Платіжна система
- Реальні аудіо та відео (плейсхолдери)
- Мультипрофільність (кілька дітей)
- Адмін-панель для контенту
- Офлайн-режим

## Відомі технічні ризики

| Ризик | Проблема | Рішення (обрати) |
|-------|----------|-----------------|
| **Зберігання даних** | localStorage стирається при очищенні кешу або зміні пристрою | Email-бекап / код відновлення / backend |
| **Контент-доставка** | 365 аудіо + 365 відео = гігабайти | CDN + API (потрібен backend) |
| **Контент-пайплайн** | Без адмін-панелі кожне оновлення = новий деплой | Headless CMS (Sanity, Contentful) або кастомна адмінка |
| **Кнопка «скинути»** | Зараз є в UI — в production її не має бути | Прибрати або перенести в прихований dev-режим |

## Технічні нотатки

- **Платформа:** веб-застосунок (браузер, mobile-first)
- **Фреймворк:** React + Vite
- **Стан MVP:** localStorage
- **Стан продукту:** потребує backend (авторизація, контент, прогрес)
- **Стилі:** CSS Modules, акварельна естетика
- **Монетизація:** SaaS підписка
- **Мова:** українська
