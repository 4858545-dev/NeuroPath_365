# docs/screens.md — Специфікація екранів НейроВітамінки 365

> Це єдина точка правди для UI розробки.  
> Claude Code має читати цей файл перед створенням будь-якого компонента екрану.

---

## Загальні правила UI

- Max-width: 420px, по центру на десктопі
- Всі екрани — вертикальний скрол якщо контент не вміщується
- Мова: українська скрізь
- Шрифт: Nunito
- Персонаж: Ростік (SproutSvg.jsx)
- Основний колір кнопок дії: `--color-primary` (зелений)
- Кнопка "далі" завжди внизу екрану, фіксована або після контенту

## 1. OnboardingScreen

**Умова показу:** `childName === null` в store.

**Крок 1 — Привітання**
- Повноекранний однотонний фон (`--color-bg`)
- По центру: анімований Ростік (підстрибує або махає)
- Бабл над Ростіком: *"Ой! Привіт! Я нарешті прокинувся. Мені потрібен друг!"*
- Знизу: велика зелена кнопка **"Познайомитись"**
- При тапі → Крок 2

**Крок 2 — Ім'я дитини**
- Однотонний фон
- Заголовок: *"Як тебе звати?"*
- Ростік збоку, менший
- Текстове поле одразу активне (автофокус), підсвічене зеленим
- Placeholder: *"Введи своє ім'я"*
- Кнопка **"Далі"** — неактивна поки поле порожнє, активується при введенні
- При тапі → Крок 3

**Крок 3 — Вік дитини**
- Заголовок: *"Скільки тобі років?"*
- 5 великих кнопок-кіл з цифрами: **3 · 4 · 5 · 6 · 7**
- При виборі — кнопка підсвічується зеленим
- Кнопка **"Починаємо пригоду!"** — неактивна до вибору, активується після
- При тапі → викликає `setChild(name, age)` → переходить на `home`

## 2. HomeScreen

**Умова показу:** `childName !== null && screen === 'home'`

**Верхня частина**
- Прогрес-стрічка: кількість пройдених днів з 6 (наприклад "День 3 з 6")
- Невелика лінійна прогрес-панель під стрічкою

**Центр**
- Ростік по центру (статичний або легка idle-анімація — погойдується)
- Бабл з іменним привітанням: *"Привіт, [ім'я]! Готовий до пригоди?"*
- Жовта велика кнопка **"Пригода дня"**
- При тапі → переходить на `lesson`, починає з `currentPhase`

**Нижня частина — "Мій чарівний садок"**
- Заголовок секції: *"Мій чарівний садок"*
- Якщо `leaves === 0`: текст *"Тут з'являться твої листочки…"* (сірий, курсив)
- Якщо `leaves > 0`: відображаються листочки (іконки) — по одному за кожен пройдений день
- Максимум 6 листочків для MVP

**Стан: день вже пройдено (`todayDone === true`)**
- Кнопка "Пригода дня" неактивна або змінюється на **"До завтра 🌙"**
- Бабл Ростіка: *"Молодець! Побачимось завтра."*

**Стан: пробний період закінчився (`trialExpired === true`)**
- Показуємо `OfferScreen` замість HomeScreen (або модальне вікно поверх)

## 3. LessonScreen

**Умова показу:** `screen === 'lesson'`

**Загальна структура для всіх фаз:**
- Вгорі: кнопка "← Назад" (повертає на HomeScreen, не скидає фазу)
- Під нею: прогрес-бар з 3 сегментів (фази 1, 2, 3)
  - Пройдені: зелені
  - Поточна: жовта
  - Майбутні: сірі
- Основний контент (залежить від фази)
- Кнопка дії внизу

---

### Фаза 1 — Казка (`type: "story"`)

**Верх:** напис *"Казка дня"*, прогрес-бар (сегмент 1 — жовтий)

**Центр:**
- Планшет/табло з текстом казки (скролиться якщо не вміщується)
- Текст читається батьком або дитиною разом з батьком

**Низ планшету:**
- Плашка аудіоплеєра
  - Якщо `audioUrl === null` → плашка прихована або показує *"Аудіо буде незабаром"*
  - Якщо `audioUrl` є → кнопка Play/Pause + прогрес-лінія

**Кнопка:** зелена **"До вправи →"**
- При тапі → `completePhase(day, 1)` → перехід до Фази 2

---

### Фаза 2 — Вправа (`type: "exercise"`)

**Верх:** напис *"Вправа"*, прогрес-бар (сегмент 1 зелений, сегмент 2 жовтий)

**Центр:**
- Заголовок вправи (з `heading`)
- Короткий опис (`description`)
- Відеоплеєр:
  - Якщо `videoUrl === null` → показуємо кроки вправи (`steps`) як нумерований список
  - Якщо `videoUrl` є → відеоплеєр з posterUrl як превью, тап для відтворення
- Ритм дихання (якщо є `rhythm`): візуальна підказка — коло що розширюється/стискається
  - Вдих: `inhaleSeconds` сек → Пауза: `holdSeconds` сек → Видих: `exhaleSeconds` сек

**Кнопка:** зелена **"Виконано ✓"**
- При тапі → `completePhase(day, 2)` → перехід до Фази 3

---

### Фаза 3 — Завдання (`type: "tasks"`)

Фаза 3 — це блок з кількох завдань (`items`), об'єднаних спільним сюжетом дня. Завдання проходяться послідовно, одне за одним.

**Верх:** напис *"Завдання"*, прогрес-бар (сегменти 1+2 зелені, сегмент 3 жовтий)

**Перед першим завданням — аудіо-сигнал:**
- Якщо є `preTaskAudioUrl` → програється автоматично
- Якщо `null` → показується текст `preTaskInstruction` на 2 секунди
- Після цього → перше завдання

**Між завданнями:**
- Коротка анімація переходу (Ростік підстрибує або з'являється новий фон)
- Міні-прогрес всередині фази: "Завдання 1 з 4" вгорі

---

#### Механіка: `tap_correct` — Обери правильну відповідь

- Питання великим шрифтом
- 2–3 кнопки-варіанти вертикально
- Правильно: кнопка зеленіє → `feedbackCorrect` → через 1.5 сек наступне завдання
- Неправильно: кнопка червоніє на 1 сек → `feedbackWrong` → спробуй ще раз (необмежено)

---

#### Механіка: `tap_sequence` — Натискай по черзі

Використовується для: рахунку, послідовних дій (пушинки відлітають).

- На екрані сцена з об'єктами (`scene.items`)
- Дитина натискає на об'єкти по черзі — кожен тап запускає анімацію (`tapEffect`)
- Лічильник збоку показує скільки вже натиснуто
- Після того як всі об'єкти натиснуті → з'являється `question` з варіантами
- Якщо є `bonus` → після основного питання з'являється бонусне

Стан помилки: якщо дитина намагається натиснути вже натиснутий об'єкт — м'яке підсвічування, нічого не відбувається.

---

#### Механіка: `drag_match` — З'єднай пари

Використовується для: логіки, зіставлення форм, тіней.

- Верхня зона: об'єкти-джерела (`sources`)
- Нижня зона: цілі-відповіді (`targets`)
- Дитина проводить пальцем від джерела до цілі — малюється лінія
- Правильне з'єднання: лінія зеленіє, пара "замикається"
- Неправильне: лінія червоніє і зникає, м'який звук
- Всі пари з'єднані → `feedbackCorrect` → наступне завдання

Важливо: лінія малюється в реальному часі поки дитина тягне палець. При підніманні пальця — перевіряється чи попала в ціль (радіус 40px).

---

#### Механіка: `tap_letter` — Знайди букву

Використовується для: фонетики, розпізнавання букв.

- На екрані кілька букв (`letters`) розкидані по сцені
- Цільова буква (`isTarget: true`) має візуальну підказку: мигає або пульсує
- Дитина натискає на потрібну букву
- Правильно → `feedbackCorrect` → якщо є `bonusQuestion` → показуємо його
- Неправильно → `feedbackWrong` → спробуй ще раз

Ціль підказки: дитина ще може не знати букву — підказка (мигання + звук) дає змогу здогадатись через асоціацію.

---

#### Механіка: `trace_letter` — Обведи контур пальцем

Використовується для: підготовки до письма, моторики.

- На екрані контур фігури або літери (`traceTarget`)
- Стартова точка підсвічена (`startPoint`)
- Напрямні точки по контуру (`guidePoints: true`) — допомагають молодшим дітям
- Дитина веде пальцем по контуру — малюється слід (`strokeColor`, `strokeWidth`)
- Завдання вважається виконаним коли дитина пройшла ≥ 80% контуру (`completionThreshold: 0.8`)
- Правильно → анімація "контур заповнюється" → `feedbackCorrect`
- Якщо відірвала палець і не дійшла до 80% → `feedbackWrong` → спробуй ще раз

Технічна нотатка: використовуємо Canvas API або SVG path з touch events. Перевірка точності — відстань від точки торкання до найближчої точки контуру ≤ 30px.

---

### Фаза 4 — Фінал дня (`type: "finale"`)

Окрема повноекранна фаза. Показується після завершення останнього завдання у Фазі 3. Прогрес-бар фаз тут не відображається.

**Верх:** прогрес-панель пройдених днів — "День X з 6" (вже оновлена через `completeToday()`)

**Центр:** Ростік великий, радісний, підстрибує (`reward.message` великим теплим шрифтом)

**Анімація:** листочок вилітає з центру екрану і плавно приземляється в панель садку внизу

**Панель нагород внизу:** відображає всі зароблені листочки. Новий листочок з'являється з анімацією одразу при відкритті фази.

**Кнопка:** велика зелена **"Завдання виконано! 🌱"**
- При тапі → `completeToday()` → перехід на `home`

## 4. OfferScreen

**Умова показу:** `trialExpired === true`

**Контент:**
- Ростік сумує або дивиться з надією
- Заголовок: *"Твій безкоштовний тиждень закінчився"*
- Підзаголовок: *"Ростік чекає на тебе в наступних пригодах!"*
- Список що отримає батько: повний доступ до 365 днів, всі казки і вправи
- Кнопка **"Продовжити пригоду"** → відкриває контакт/форму оплати (для MVP — посилання на Telegram або email)
- Дрібний текст знизу: *"Перші 6 днів — безкоштовно. Без автосписання."*

---

## 5. Логіка відновлення після виходу

**При кожному відкритті застосунку:**
1. Викликається `checkNewDay()` — перевіряє чи настав новий день
2. Якщо `todayDone === false` і `currentPhase > 0` → LessonScreen відкривається з `currentPhase`
3. Якщо `todayDone === true` → HomeScreen з заблокованою кнопкою

**Кнопка "← Назад" в LessonScreen:**
- Повертає на HomeScreen
- `currentPhase` НЕ скидається — зберігається в store
- При повторному тапі "Пригода дня" → продовжує з тієї ж фази

**Скидання фази:**
- Тільки після `completeToday()` (день пройдено)
- Або після `checkNewDay()` що виявив новий день
