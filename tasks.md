## ❓ Відкриті питання

### Вирішено
- [x] **Пропущений день** → Дозволити проходити пропущені, але не більше 2–3 за раз. При вході показати: «Ми сумували за тобою. Хочеш пройти пропущений день?»
- [x] **Повернення до пройдених днів** → Так, можна переглядати повторно
- [x] **Карта днів** → Так, є карта/стежка пригод; поточний день підсвічений; наступний відкривається після завершення попереднього
- [x] **Монетизація** → 7 днів безкоштовно → підписка 1 / 6 / 12 місяців

### Потребують відповіді
- 🔴 **Кілька днів підряд** — чи можна проходити нові дні без ліміту?
  - Варіант 1: тільки 1 новий день на добу
  - Варіант 2: можна пройти всі пропущені без обмежень
  - Варіант 3: 1 новий + пропущені (до 2–3) відкриті ✅ _схоже, цей_

- 🔴 **Вихід посередині дня** — якщо вибило або вийшов: починати день з початку чи з місця зупинки?

- 🟡 **Завдання/інтерактив** — чи є в MVP або тільки в наступній версії? (У геймдизайні це 3-й елемент кожного дня поряд з казкою та вправою)

- 🟡 **Зворотній зв'язок у завданнях** — як реагує застосунок на правильну/неправильну відповідь?

- 🟡 **Вік дитини** — дані зберігаються, але чи адаптується контент? (3 vs 7 років — різний рівень)

- 🟢 **Нагороди** — листочки можна на щось міняти або вони тільки декоративні?

- 🟢 **Що після 365-го дня?** — заклик на продовження / нова програма / велике фінальне відео?

- 🟢 **Типи казок** — 5 видів (медитативні, терапевтичні та ін.); коли «день відпочинку» — кожен 7-й?

## 🟡 MVP

### Онбординг та запуск

- [ ] Стартове відео (7–10 сек): нічне небо → дощ → Паросток проростає → бабл «Ой! Привіт!»
- [ ] **Фаза 0 (реєстрація)**: однотонний фон, Паросток з фразою, поле імені (одразу активне), 5 кнопок віку (3/4/5/6/7), велика кнопка «Починаємо пригоду!»
- [ ] Зберегти ім'я та вік в localStorage

### Головний екран (початок нового дня)

- [ ] Іменне привітання: «Привіт, [ім'я]! Готовий до пригоди?»
- [ ] Прогрес-стрічка з кількістю пройдених днів
- [ ] Паросток по центру (анімація вітання)
- [ ] Велика жовта кнопка «Пригода дня»
- [ ] Панель «Мій чарівний садок» внизу:
  - якщо нагород немає → «Тут з'являться твої листочки…»
  - якщо є → відображати листочки (кількість = пройдені дні)
- [ ] Якщо є пропущені дні → показати підказку «Ми сумували за тобою. Хочеш пройти пропущений день?»

### Екран пригоди (3 фази)

**Фаза 1 — Казка:**
- [ ] Однотонний фон, напис «Казка дня» вгорі
- [ ] Прогрес-панель (3 частини, підсвічена 1-а — жовта)
- [ ] Планшет/табло з текстом казки
- [ ] Аудіоплеєр-заглушка внизу планшета (тап → вмикається аудіо, 2–3 хв)
- [ ] Зелена кнопка «До вправи» внизу

**Фаза 2 — Відео-вправа:**
- [ ] Однотонний фон, напис «Вправа» вгорі
- [ ] Прогрес-панель (1-а — зелена, 2-а — жовта)
- [ ] Табло з відео-заглушкою + опис вправи
- [ ] Тап на трикутник-плеєр → запускається відео (1.5–3 хв)
- [ ] Зелена кнопка «Виконано» внизу

**Фаза 3 — Завершення дня:**
- [ ] Однотонний фон, напис «Готово» вгорі
- [ ] Прогрес-панель (1-а і 2-а — зелені, 3-я — жовта)
- [ ] Паросток по центру, нагорода за день
- [ ] Панель нагород: +1 листочок додається
- [ ] Пояснення для батьків: що саме прокачали та чому це важливо
- [ ] Велика кнопка «Завдання виконано» → повернення на головний екран

### Технічне налаштування

- [ ] Структура проекту: `src/screens/`, `src/components/`, `src/data/`, `src/utils/`
- [ ] CSS-змінні: зелена/жовта/м'яка палітра, акварельна естетика
- [ ] Базовий роутинг між екранами (state-based, без бібліотеки)
- [ ] localStorage утиліти: `getProgress`, `setProgress`, `resetProgress`
- [ ] Контент-схема дня: `{ id, title, storyText, audioUrl, videoUrl, rewardLeaf }`
- [ ] Плейсхолдер-контент для дня 1
- [ ] Логіка пропущених днів (показати підказку, дозволити пройти до 2–3)
- [ ] Прибрати або сховати в dev-режим кнопку «скинути прогрес»
- [ ] Перевірка на 375px та 414px (mobile-first)
